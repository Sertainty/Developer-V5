ACTIVE · INTELLIGENT · DATA

Data that protects itself

The Sertainty UXP Technology embeds active intelligence inside a protected object, preventing unauthorized access no matter where the data resides. Unlike conventional encryption, it constructs dynamic protection schemes that resist quantum systems and AI data-mining engines — securing data at rest, in flight, and in process.

Three Core Objects
UXP Identity · .iic

Who can open

Authenticates a user, machine, or workgroup via challenge/response. Your credentials — treat like a private key, never share.

UXP Object · .uxp

What is protected

A self-enforcing container holding files, folders, or SQL data behind an embedded policy. The payload — safe to share anywhere.

SmartMessage

Peer-to-peer payload

A protected in-memory or on-disk message for direct exchange between identities — no containing file required.

Key Capabilities
Selective & partial access
  • Who can read what — selective access governs page-level reach.
  • How much is unprotected — only the portion in use, at runtime.
  • How long plaintext exists — decrypted just in time, released after use.
Active Intelligence, Active Authorization

Every open triggers a fresh evaluation against the embedded rules: identity, time, location, device, and prior failures. Access is never unconditional.

UXP SQL

SQLite-compatible relational data lives inside the UXP, with every query gated by the same Identity and rules that protect the file payload.

Identity embedded at creation

No external key store. The access policy is compiled into the UXP object at creation and travels with it.

Offline by default. Airgap ready.
  • No server, key manager, or network is needed for access decisions.
  • UXP runs identically in connected and airgapped environments.
  • Network features are opt-in.
Audit log & SIEM

Logging is a preference set by the data owner. When the UXP is online, access events can be sent to a server through several outbound methods, ready for ingest by a SIEM or other monitoring tool. Storing the audit log inside the UXP itself is also an option.

Self-destruct & Alternate Reality

Configurable defensive responses when access conditions fail: shred the contents, expose decoy data, or revoke the file silently.

Delegate access without re-protecting

Authorize and revoke users dynamically through the Delegate Service, without rebuilding or redistributing the UXP.
